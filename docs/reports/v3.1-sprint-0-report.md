# Sprint 0 완료 보고서

## 개요
- **Sprint**: 0 — 모바일 UI 최적화
- **버전**: v3.1
- **커밋**: (여러 커밋에 걸쳐 구현)
- **상태**: 완료

## 목표
게임 중 과도하게 표시되던 UI 프레임을 정리하고, 모바일 환경에서 터치 타겟과 HUD 레이아웃을 최적화한다. UIVisibilityManager를 신설해 lobby/running/minigame/paused 상태별 UI를 일괄 제어하며, iPhone SE 해상도에서도 겹침 없는 HUD를 보장한다.

## 완료 항목
- UIVisibilityManager 모듈 신설 — lobby/running/minigame/paused 상태별 UI 일괄 제어
- DailyChallengeController 게임 중 숨김 처리
- ObjectiveHudController 아이콘 축소 모드 (3초 후 자동 축소, 탭 시 1.5초 펼침)
- ProgressionController LevelContainer + MissionButton 게임 중 Visible=false
- RunSkillHudController XP 바 게임 중 숨김 처리
- HUD 레이아웃 재배치 — ActiveSkill 버튼 우측 하단으로 이동
- UITokens.Touch.MIN_TARGET_SIZE = 48 상수 추가, 전 버튼 터치 타겟 48px 검증
- ResponsiveUtil compactMode 적용 (500px 미만 화면에서 HUD 추가 축소)
- 스모크 테스트 체크리스트 작성 (`docs/test/v3.1-sprint0-smoke-test.md`)
