# Coin Runner - 게임 기획서 v1.0

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **게임명** | Coin Runner (코인 러너) |
| **장르** | 횡스크롤 코인 수집 러너 |
| **타겟 연령** | 6~12세 |
| **플레이 인원** | 1인 (싱글 플레이) |
| **플랫폼** | Roblox (PC, 모바일, 태블릿) |
| **세션 시간** | 1회 플레이 2~5분 |

## 2. 게임 컨셉

> "뛰고, 점프하고, 코인을 모아라!"

플레이어 캐릭터가 자동으로 달리며, 장애물을 피하고 코인을 수집하는 간단한 횡스크롤 러너 게임.
저연령층이 복잡한 조작 없이 **점프**와 **슬라이드** 두 가지 동작만으로 즐길 수 있다.

## 3. 핵심 게임플레이

### 3.1 조작 방법

| 조작 | PC | 모바일 |
|------|-----|--------|
| **점프** | Space / W / ↑ | 화면 오른쪽 탭 |
| **슬라이드** | S / ↓ | 화면 왼쪽 탭 |

- 2단 점프: 공중에서 한 번 더 점프 가능 (높은 코인 수집용)
- 슬라이드: 낮은 장애물 통과, 일정 시간 후 자동 복귀

### 3.2 기본 메카닉

```
[자동 달리기] → [장애물 회피] → [코인 수집] → [점수 획득]
         ↓                                    ↓
    [속도 점진 증가]                     [상점에서 아이템 구매]
```

- **자동 달리기**: 캐릭터는 자동으로 오른쪽으로 달린다
- **속도 증가**: 시간이 지날수록 서서히 속도가 증가한다 (난이도 상승)
- **코인 수집**: 경로에 배치된 코인을 터치하면 자동 수집
- **장애물**: 부딪히면 라이프 1개 소모
- **라이프**: 기본 3개, 모두 소진 시 게임 오버

### 3.3 코인 종류

| 코인 | 색상 | 점수 | 출현 빈도 |
|------|------|------|-----------|
| **브론즈 코인** | 갈색 | 1점 | 매우 높음 |
| **실버 코인** | 은색 | 5점 | 보통 |
| **골드 코인** | 금색 | 10점 | 낮음 |
| **다이아 코인** | 파란색 빛남 | 50점 | 매우 낮음 |

### 3.4 장애물 종류

| 장애물 | 회피 방법 | 설명 |
|--------|-----------|------|
| **낮은 벽** | 점프 | 작은 블록 장애물 |
| **높은 벽** | 슬라이드 | 위쪽이 막힌 터널형 |
| **구덩이** | 점프 | 바닥이 빈 구간 |
| **움직이는 벽** | 타이밍 점프 | 위아래로 움직이는 장애물 (후반부) |

### 3.5 파워업 아이템

| 아이템 | 효과 | 지속시간 |
|--------|------|----------|
| **자석** | 주변 코인을 자동 흡수 | 5초 |
| **쉴드** | 장애물 1회 무시 | 1회 |
| **2x 코인** | 코인 점수 2배 | 8초 |
| **미니 사이즈** | 캐릭터 축소, 장애물 쉽게 회피 | 5초 |

## 4. 게임 흐름

```
[로비 화면]
    ├── [플레이] → [카운트다운 3,2,1] → [게임 시작]
    │       └── [게임 오버] → [결과 화면] → [로비]
    ├── [상점] → [캐릭터/펫/이펙트 구매]
    ├── [컬렉션] → [수집한 아이템 확인]
    └── [리더보드] → [하이스코어 확인]
```

### 4.1 로비 화면

- 심플한 메인 화면: 큰 "PLAY" 버튼
- 현재 보유 코인 표시
- 장착 중인 캐릭터/펫 미리보기
- 하단 메뉴: 상점, 컬렉션, 리더보드

### 4.2 게임 플레이 화면

- 상단 좌측: 현재 점수
- 상단 우측: 보유 라이프 (하트 아이콘)
- 상단 중앙: 현재 코인 수
- 하단: 모바일 조작 버튼 (모바일만)
- 활성 파워업 아이콘 표시

### 4.3 게임 오버 화면

- 획득 점수 / 최고 기록 비교
- 획득 코인 수
- "다시하기" / "로비" 버튼
- **부활 기회** (광고 또는 Robux) - 1회 제한

## 5. 월드 시스템

### 5.1 테마 월드 (향후 확장 가능)

| 월드 | 테마 | 특징 |
|------|------|------|
| **월드 1** | 초원 | 기본 스테이지, 밝은 색감, 쉬운 난이도 |
| **월드 2** | 도시 (잠금)| 빌딩 배경, 중간 난이도 |
| **월드 3** | 우주 (잠금)| 낮은 중력, 높은 난이도 |

- **v1.0에서는 월드 1 (초원)만 구현**
- 이후 업데이트로 월드 추가

### 5.2 맵 구성 방식

- **청크 기반 무한 생성**: 미리 만든 맵 청크를 랜덤으로 이어붙이는 방식
- 청크 난이도 단계: Easy / Normal / Hard
- 게임 진행 시간에 따라 높은 난이도 청크 출현 비율 증가
- 지나간 청크는 자동 삭제 (메모리 관리)

## 6. 경제 시스템 & 수익화

### 6.1 재화

| 재화 | 획득 방법 | 용도 |
|------|-----------|------|
| **코인 (무료)** | 게임 플레이 | 기본 아이템 구매 |
| **젬 (프리미엄)** | Robux 구매, 업적 보상 | 프리미엄 아이템, 부활 |

### 6.2 Robux 상품 (수익화)

아동 친화적이고 부담 없는 가격대:

| 상품 | 가격 (Robux) | 내용 |
|------|-------------|------|
| **스타터 팩** | 49 | 젬 100 + 랜덤 펫 1마리 |
| **젬 소량** | 25 | 젬 50 |
| **젬 중량** | 75 | 젬 200 (보너스 +50) |
| **젬 대량** | 149 | 젬 500 (보너스 +150) |
| **VIP 패스** | 99 | 코인 2배 영구 + VIP 뱃지 |

### 6.3 상점 아이템

#### 캐릭터 스킨

| 스킨 | 가격 | 재화 |
|------|------|------|
| 기본 캐릭터 | 무료 | - |
| 닌자 | 500 코인 | 코인 |
| 로봇 | 1000 코인 | 코인 |
| 유니콘 | 100 젬 | 젬 |
| 드래곤 | 200 젬 | 젬 |

#### 펫 (따라다니는 동반자)

| 펫 | 효과 | 가격 |
|----|------|------|
| 강아지 | 코인 +5% 보너스 | 300 코인 |
| 고양이 | 자석 범위 +20% | 500 코인 |
| 피닉스 | 코인 +10% 보너스 | 150 젬 |
| 드래곤 | 쉴드 시작 | 250 젬 |

#### 트레일 이펙트 (달릴 때 뒤에 남는 효과)

| 이펙트 | 가격 | 재화 |
|--------|------|------|
| 별가루 | 200 코인 | 코인 |
| 무지개 | 400 코인 | 코인 |
| 불꽃 | 80 젬 | 젬 |
| 네온 | 120 젬 | 젬 |

### 6.4 수익화 원칙

- **Pay-to-Win 금지**: 유료 아이템은 외형(코스메틱) 위주
- **펫 효과는 미미하게**: 보너스를 5~10% 수준으로 제한
- **모든 아이템은 무료 재화로도 획득 가능** (시간 투자 필요)
- **사행성 요소 최소화**: 가챠/랜덤 박스 지양, 직접 구매 방식
- **아동 보호**: 과도한 구매 유도 UI 금지

## 7. 진행 시스템

### 7.1 플레이어 레벨

| 레벨 | 필요 경험치 | 보상 |
|------|-------------|------|
| 1 | 0 | 시작 |
| 2 | 100 | 젬 5 |
| 3 | 250 | 닌자 스킨 해금 |
| 5 | 600 | 자석 파워업 해금 |
| 10 | 2000 | 젬 20 + 랜덤 트레일 |
| 20 | 5000 | 프리미엄 칭호 |

- 경험치 = 획득 점수의 일정 비율
- 레벨업 시 팡파르 연출 + 보상

| 미션 | 보상 |
|------|------|

| 코인 100개 모으기 | 코인 30 |
| 점수 500점 달성하기 | 젬 3 |
| 장애물 50개 피하기 | 코인 80 |

### 7.3 업적

| 업적 | 조건 | 보상 |
|------|------|------|
| 첫 걸음 | 첫 게임 클리어 | 코인 100 |

## 8. 소셜 기능

### 8.1 리더보드

- **일간 / 주간 / 역대** 하이스코어 보드
- 상위 100명 표시
- 주간 1위에게 특별 보상 (젬 50)

### 8.2 향후 확장 (v2.0+)

- 길드 시스템
- 시즌 패스

## 9. 기술 사양

### 9.1 카메라

- 횡스크롤 시점: 캐릭터를 화면 왼쪽 1/3 지점에 고정
- 카메라가 캐릭터를 따라 오른쪽으로 이동
- 약간의 Look-ahead (캐릭터 진행 방향으로 카메라 약간 앞서기)

### 9.2 성능 목표

- 60 FPS 유지 (모바일 포함)
- 동시 렌더링 청크: 최대 3개
- 지나간 청크 즉시 Destroy
- 파티클 이펙트 최소화 (모바일 고려)

### 9.3 데이터 저장

- **DataStore 사용**: 플레이어 데이터 서버 저장
- 저장 항목: 코인, 젬, 레벨, 경험치, 보유 아이템, 하이스코어, 일일 미션 진행도
