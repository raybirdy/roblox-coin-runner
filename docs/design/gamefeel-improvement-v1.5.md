# 게임 매끄러움 개선 제안서 v1.5
**작성일**: 2026-03-05
**참여 전문가**: 레벨 디자이너(LD) · 게임 필 디자이너(GFD) · 비주얼 디렉터(VD)
**상태**: 전문가 만장일치 합의 완료 ✅

---

## 토론 배경

유저 피드백:
> "게임 진행이 여전히 매끄럽지 않아. 코인의 위치, 코인 수집 시의 이펙트(타격감이 없어.), 장애물의 종류.. fever 타임 시 별 차이가 없어."

4개 영역에 대해 전문가 3인이 코드베이스 직접 분석 후 원탁 토론을 진행하였습니다.

---

## 전문가 분석 요약

### 영역 1: 코인 수집 타격감 (게임 필 디자이너)

**근본 원인 진단:**
| 문제 | 현재 코드 | 증상 |
|------|-----------|------|
| 카메라 반응 없음 | 일반 코인 수집 시 Shake() 미호출 | "뭔가 먹은 느낌"이 없음 |
| 코인 즉시 소멸 | `Transparency = 1` 즉시 처리 | 획득 순간이 인식되지 않음 |
| HUD 무반응 | `UpdateCoins()`가 텍스트만 교체 | 점수 증가가 "안 느껴짐" |
| 파티클이 느리고 둥둥 | Lifetime 0.5~1.0s, Speed 5~15, 위로 표류 | "구름" 같은 느낌, 임팩트 없음 |
| 단일 사운드 인스턴스 | 연속 수집 시 사운드가 씹힘 | 빠른 수집의 쾌감이 없음 |
| Highlight Occluded | `DepthMode = Occluded` | 캐릭터 플래시가 거의 안 보임 |

---

### 영역 2: 코인 배치 & 장애물 (레벨 디자이너)

**근본 원인 진단:**
| 문제 | 현재 코드 | 증상 |
|------|-----------|------|
| 코인과 장애물이 독립적 | 각자 랜덤 배치 | "코인이 동선 안내를 안 함" |
| 균등 밀도 배치 | 청크 전체에 균등 분산 | 긴장-이완 리듬 없음 |
| 장애물 조합 없음 | 이전 장애물 타입 미기억 | 단조로운 반복 |
| 청크 간 연속성 미확보 | 청크 독립 생성 | 경계에서 불공정한 죽음 발생 |
| 슬라이드 전용 장애물 1종뿐 | Tunnel만 슬라이드 | 점프:슬라이드 = 5:1 불균형 |

---

### 영역 3: 피버 시각 차별화 (비주얼 디렉터)

**근본 원인 진단:**
| 문제 | 현재 코드 | 증상 |
|------|-----------|------|
| 오버레이 85% 투명 | 0.3초 후 85% 투명으로 페이드 | 거의 안 보임 |
| 카메라 줌인 없음 | 피버 시 `Zoom()` 미호출 | 속도감 변화 미미 |
| 포스트 프로세싱 없음 | BloomEffect/ColorCorrection 미사용 | 화면 전체 변화 없음 |
| "FEVER!" 텍스트 1.5초 후 사라짐 | 퍼시스턴트 배너 없음 | 피버 지속 중인지 잊음 |
| 파티클 피버와 일반 동일 | SpeedEffectController 피버 인식 없음 | 시각적 차별화 없음 |

## 전문가 합의 선언

> **레벨 디자이너 (LD): ✅ Y**
> "코인 밀도 리듬(P1-7)과 청크 간 연속성(P1-8)이 포함되어, 레벨 디자인의 근본적인 긴장-이완 구조가 만들어진다. 불공정한 죽음 제거는 6~12세 타깃에서 이탈률을 직접적으로 낮출 핵심 변경이다. 이 10개면 '코인 모으는 재미'의 질적 도약이 가능하다."

> **게임 필 디자이너 (GFD): ✅ Y**
> "코인 수집 한 번의 경험이 시각(파티클 팡! + 스케일팝) + 청각(상승 음계) + 촉각(카메라 쉐이크) 3감각 동시 자극으로 변한다. 러너 게임의 핵심은 '반복되는 작은 보상이 매번 기분 좋은가'이고, 이 10개 변경은 그 질문에 명확한 예를 만든다."

> **비주얼 디렉터 (VD): ✅ Y**
> "피버 포스트 프로세싱(P1-9)과 카메라 줌인(P0-4)이 합쳐지면, 피버 모드가 화면 전체가 빛나며 캐릭터가 가까워지는 특별한 순간이 된다. P1까지 완성하면 상용 러너 게임 수준의 연출이 된다."
