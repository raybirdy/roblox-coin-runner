## 2. 핵심 문제 진단

### 문제 1: 세션 길이 목표 미달 (2~5분 → 목표 12분)

**근본 원인:**
- 속도가 120초 만에 최대 52 studs/s에 도달 후 정체
- 장애물 밀도 단조 증가 → 어린 유저 이탈 가속
- Phase Shift 없음: 동일한 메커닉 반복 (점프/슬라이드뿐)

**해결책: 난이도 호흡 패턴 + 챌린지 웨이브**

현재 `CHUNK_SPAWN_RATES`는 단조 증가. 교체안:

```
구조: 하드 구간 (15초) → 이지 구간 (10초) → 반복
Zone 전환 시: 속도 15% 일시 감소 (호흡 구간)
매 75~90초: "스타 웨이브" (장애물 없음, 코인 패턴만)
```

### 문제 2: 코인 인플레이션 — 코인 싱크 부족

**현재 코인 수지:**
- 일 수입 (중급, 6판): ~1,100 코인
- 일 지출 (평균): ~300 코인
- **잉여: ~800 코인/일**

장기 유저는 만렙 + 모든 코인 스킨/업그레이드 달성 후 50,000+ 코인 쌓임 → 젬 구매 동기 감소.

**해결책:**
1. **펫 시스템** (최우선): 6종 × 5레벨 = 코인 15,000~60,000 추가 싱크
2. **일일 회전 코인 상점** (200~500코인 소모성 임시 효과)
3. **업그레이드 프레스티지** (최대 레벨 달성 후 코인 2배 투자로 리셋 + 코스메틱 뱃지)

### 문제 3: VIP vs 구독 포지셔닝 충돌

VIP GamePass (영구): 2배 코인 멀티플라이어
구독 (월 99 Robux): 일 5젬 + 2배 XP + 스킨 + 광고 없음

**문제**: VIP로 코인 문제를 해결한 유저에게 구독의 매력이 떨어짐.

**해결책**: VIP를 코스메틱/소셜 패스로 전환
- VIP 전용 로비 구역, VIP 전용 스킨 2종, VIP 뱃지
- 2배 코인 멀티플라이어를 VIP에서 제거하고 **구독 혜택**으로 이동

### 문제 4: 위협 요소 부재

Temple Run (악마 원숭이), Subway Surfers (경찰+개)처럼 **플레이어를 쫓는 존재**가 없음.
6~12세에게 "쫓기는 긴박감"은 가장 강력한 몰입 메커닉.

**해결책: 친근한 경쟁자 러너 (Ghost Rival)**
- 기존 로비 NPC 캐릭터(토끼/곰돌이) 중 하나가 플레이어 옆에서 함께 달림
- 플레이어가 피격/실수 시 경쟁자가 앞서나감 → "따라잡아야 한다"는 동기
- 경쟁자 속도는 플레이어 현재 퍼스널 베스트 기록 기준 (달성 가능한 도전)

## 3. 우선순위별 개선안

### Tier S: 즉시 구현 (세션 길이 +3~5분 기대)

#### S1. 난이도 호흡 패턴
**대상 파일**: `GameConstants.luau` (CHUNK_SPAWN_RATES), `ChunkService.luau`
**변경 규모**: 소

Zone 전환마다 Easy 청크 2개 삽입 → 미니 안전 구간
`CHUNK_SPAWN_RATES`를 시간 구간 사이클 대신 Zone 기반으로 재구성

#### S2. 챌린지 웨이브 (매 75~90초)
**대상 파일**: 새 `ChallengeWaveService.luau`, `GameManager.luau`, `ChunkService.luau`
**변경 규모**: 대

```
발동: 75초마다 or Zone 전환 시
내용: 15~20초간 장애물 없음, 코인 패턴 폭발
목표: "80% 이상 코인 수집!" (진행 바 표시)
성공 보상: +1 라이프 (최대 3개 이하 시), 피버 게이지 30% 충전
실패 효과 없음 (저연령 배려)
```

#### S3. 파워업 리스폰 주기 단축
**대상 파일**: `GameConstants.luau` (POWERUP)
**변경 규모**: 최소

```lua
MIN_SPAWN_TIME = 20 → 12  -- 첫 파워업 더 빠르게
MIN_INTERVAL   = 15 → 10  -- 파워업 더 자주
-- 30% → 38% 출현 확률 (Magnet)
```

## 7. 기술 부채 및 아키텍처 메모

### GameSession 타입 성장 문제
현재 `GameSession` 타입은 40+개 필드. 관련 필드를 서브테이블로 그룹화 권장:
```lua
type GameSession = {
    ...
    feverState: { isActive: boolean, isSuperFever: boolean, endTime: number, ... },
    survivalState: { multiplierTier: number, lastHitTime: number, hitRushActive: boolean, ... },
    battleState: { asyncBattleTarget: any, realtimeOpponent: any, ... },
}
```

### BossRunService 활용
기존 `BossRunService.luau`와 `BossRunController.luau`가 유휴 상태.
Ghost Rival 시스템 구현에 이 서비스 재활용 가능.

### ObjectiveService 확장
새 조건 타입 (`consecutive_silver`, `no_slide_distance`, `hit_rush_count`, `near_miss_chain`, `zone_reached`) 을 `getProgressValue()`에 추가 필요.
각 세션에서 해당 데이터 추적도 `GameManager` heartbeat에 추가.
